## バグ詳細

### BUG-001: `\n` リテラル表示 [High]
**場所**: 相談詳細ページ、相談一覧プレビュー
**症状**: 本文中の `\n` が改行として解釈されず、文字列 `\n\n` がそのまま画面に表示される
**再現**: https://morimichi.cc/consultations/259352c5-5bc0-4078-9b21-04f5db4c111c
**修正案**: 本文表示コンポーネントに `whitespace-pre-wrap` を適用、またはテキスト保存時に改行コードを正規化

### BUG-002: 自己リアクション 500 エラー [High]
**場所**: `POST /api/reactions`
**症状**: 自分の相談への共感クリックで 500 `create_failed` が返る
**期待**: 400 `cannot_react_own` またはUIで共感ボタンを無効化
**原因推定**: サーバーサイドの `supabase.auth.getUser()` が Cookie セッションを正しく読めず、`user.id` とBDの `user_id` の比較がスキップされ、RLS で INSERT がブロックされて 500 になる
**影響**: ユーザー体験に問題（無言のエラー）。ただしデータ整合性は保たれている（RLSが最終防壁として機能）

### BUG-003: コンタクトリクエスト送信 500 [High]
**場所**: `POST /api/contact-requests`
**症状**: 「運営を通じてこの人に相談する」フォーム送信で 500 エラー
**原因推定**: BUG-002 と同根 — サーバーサイドセッション読み取りの問題。`createServerSupabaseClient()` が Cookie からユーザーを特定できていない可能性
**注**: REST API で直接ログイン → `/api/auth/session` で Cookie 同期した場合の固有問題の可能性もあり。通常のフォームログイン経由では正常動作する可能性がある。

### BUG-004: React Hydration Mismatch [Medium]
**場所**: 相談詳細ページ
**症状**: コンソールに `Minified React error #418` が繰り返し記録される
**影響**: 機能的な問題は軽微だが、パフォーマンスとデバッグに影響
**原因推定**: SSR時とCSR時でテキストコンテンツが異なる（日時フォーマット等の可能性）
